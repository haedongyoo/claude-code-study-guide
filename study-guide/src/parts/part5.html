Part 5: Role-Based Playbooks

Chapters 7–8

Product managers, strategists, and founders share a common starting point: organizing research and making decisions quickly. The first wins come from research briefs, meeting note structuring, and decision documentation — not from full automation.

💡 Pro Tip: For PMs, the ideal Claude adoption path is: (1) Meeting note summaries → (2) Weekly research briefs → (3) PRD draft generation → (4) Competitive analysis with templates. Each step builds on the previous file structure.

Founder / Solo Operator minimum file set:

workspace/
├── about-me.md          # Background, industry, goals
├── working-rules.md     # Tone, forbidden topics, priorities
├── projects/
│   └── current/
│       ├── brief.md     # This week's focus
│       └── competitors.md
├── templates/
│   └── weekly-brief.md
└── outputs/
📌 Key Concept: For strategists, the human must always own: market interpretation, priority decisions, and stakeholder alignment. Claude accelerates the research and structuring — not the judgment.

What humans must keep: Market interpretation, priority setting, and stakeholder negotiation. Claude handles research gathering, draft structuring, and format consistency.

Marketing teams get the fastest returns from Claude when they establish brand voice files and content templates first. Without brand-voice.md and anti-ai-writing-style.md, every piece of content requires extensive tone correction.

Content operations adoption path:

  1. Topic research and content structure outlines
  2. First drafts with brand voice templates
  3. Multi-channel adaptation (blog → social → newsletter)
  4. SEO optimization with structured templates
  5. Regular content calendar automation
⚠️ Common Mistake: Saying "write in our company tone" without providing a brand-voice.md. Claude cannot intuit your brand. Create a file with: preferred phrases, forbidden phrases, sentence length guidelines, formality level, and 2–3 example paragraphs showing the ideal tone.

What humans must keep: Final message decisions, brand tone approval, and external publication sign-off. Claude handles research synthesis, structural drafting, and format adaptation.

Sales and customer success teams benefit most from account briefs and follow-up email drafts. The key constraint: all external communications must pass through human approval gates.

Operations and back-office automation should start with the lowest-risk tasks: folder organization, classification, draft summaries, and template-based reports. High-risk automation (actual data entry, payment processing, deletion) should only come after governance structures are in place.

📌 Key Concept: For agency and team operators, the critical principle is client context isolation. When managing multiple clients simultaneously, the biggest risk is not slow output — it is context contamination between clients. Separate folders per client, shared templates, and strict naming conventions prevent cross-client leakage. 
clients/
├── client-a/
│   ├── brief.md
│   ├── glossary.md
│   └── approval-rules.md
├── client-b/
│   ├── brief.md
│   └── ...
templates/
├── client-report-template.md
└── review-checklist.md
team/
├── shared-rules.md
└── naming-convention.md
💡 Pro Tip: In an agency context, one folder boundary prevents more accidents than one clever prompt. Speed matters less than making sure Client A's data never leaks into Client B's output.

Design/Branding: Designers get the most value from reference-based exploration and implementation QA. Use images as input — screenshots of bugs, reference layouts, desired states. Visual context often communicates what paragraphs of CSS descriptions cannot.

Engineering: The strongest first task is always a small bug fix with a test, not a large new feature. The adoption path: (1) bug fix + test → (2) refactoring with plan → (3) code review with review split → (4) parallel worktree experiments → (5) CI integration and automated hooks.

📌 Key Concept: For researchers and analysts, Claude excels at: summarization, glossary creation, comparison tables, question generation, and structured literature reviews. The human must always own: claim interpretation, reproduction result judgment, and original insight.

Adoption priority across all roles follows the same pattern:

  1. Gather and organize information (lowest risk)
  2. Create first drafts with templates
  3. Review and refine with verification loops
  4. Automate recurring workflows with skills
  5. Scale with parallel agents and scheduled tasks (highest risk, last step)
⚠️ Common Mistake: Starting with end-to-end automation. The teams that go farthest start with the front-end of the process (organizing, structuring, drafting) and only automate the back-end (execution, deployment, delivery) after governance is established.

Chapter 8 is not a feature encyclopedia — it is a situation matching table. The approach: identify today's blocker → classify the task shape → select the file set → execute → verify through approval gate → leave a structured handoff.

The workflow for any blocked scenario:

  1. Today's blocker — What specific task is stuck?
  2. Task shape — Is it research, drafting, implementation, or review?
  3. File set selection — Which minimum files does this task need?
  4. Execution — Run the workflow with output contract
  5. Approval gate — Human checks before anything goes external
  6. Structured handoff — State and next steps for the next session
📌 Key Concept: Key terms for scenario matching: output contract (what format and location results must follow), token hotspot (where context suddenly gets expensive), approval gate (mandatory human check before external delivery), and structured handoff (formatted state memo for session continuity).
💡 Pro Tip: The gap ahead will widen not between people who know more features, but between teams that learn role-specific delegation strategies faster. Development and customer support adopt first (broad, low-risk tasks); legal, finance, and research adopt more cautiously (summary and classification first, judgment later).
