Part 1: Building Fundamentals

Chapters 1–2

The AI landscape shifted from "which model is smarter" to "which tool connects files, tests, Git, browser, and external services with the least friction." This is why Claude Code, Codex, and Gemini CLI are discussed together — the competition is now about work paths, not benchmark scores.

📌 Key Concept: Claude Code is not just a coding app. It is a work harness combining memoized context, tool orchestration, permission gates, and resumable sessions. Cowork, Skills, Plugins, and Scheduled Tasks are extensions of this core harness into non-coding work.

Research confirms this direction. METR reports that agent task horizons are growing rapidly, and Anthropic notes that the real performance gap comes from task structure and verification habits, not model names. The question is no longer "how smart is the model?" but "who learns delegation rules and verification harnesses faster?"

💡 Pro Tip: Start by asking: "Is my task question-based, document-based, or implementation-based?" This single question determines whether you should use Chat, Projects, Cowork, or Claude Code.

Before diving deeper, internalize these four terms. Every chapter builds on them.

TermMeaning
Memoized ContextBackground info held across turns so you don't re-explain every time
Tool OrchestrationThe layer that decides which tools to use, in what order
Permission GateThe boundary that separates allow, ask, and deny before execution
Resumable SessionThe ability to pick up where you left off, even after disconnection

These form a pipeline: User Request → Memoized Context → Tool Orchestration → Permission Gate → Execution & Verification → Resumable Session → Next Task. No matter how complex a workflow gets, it is always a variation of these five stages.

📌 Key Concept: "Memoized" doesn't mean "memorized." It means "computed once, reused many times" — like setting background rules that persist across every turn without re-sending them.

A harness is the outer structure that defines what Claude can read, modify, and verify. Think of it as setting up a colleague's workspace: you place reference materials on the desk, decide which drawers are accessible, and attach a checklist to complete before finishing.

⚠️ Common Mistake: Sending "Fix the login bug" with no boundaries. A strong harness request looks like: "Read only auth/, don't touch .env, pass login tests, leave a 3-line change note."

Harness Engineering is not about writing smarter prompts — it is about designing the workspace where Claude operates. It includes: which files are always loaded, what permissions are set, what tests must pass, and where results are stored.

The International AI Safety Report 2026 confirms: as model capabilities rise, harness and verification structures become more important, not less. Real-world tasks involve resource constraints, incomplete information, exceptions, and approval flows that raw model intelligence cannot handle alone.

💡 Pro Tip: Even beginners can start building a harness: write basic rules in CLAUDE.md, set allow/deny scopes in settings.json, split big tasks with plan.md, and leave session state in handoff.md.

The same Claude model behaves differently depending on which surface you use. Choosing the wrong surface is not a skill problem — it is a routing problem.

SurfaceBest ForInfluenced By
ChatQuick questions, idea sortingQuestion quality
ProjectsPersistent background knowledgeFolder structure, templates
CoworkDocument delegation — research, briefs, file organizationFolder structure, templates
Claude CodeImplementation and verification — code, tests, reviewsRule files, verification loops
⚠️ Common Mistake: Three misconceptions: (1) "Same model means same experience everywhere" — wrong, the surface matters enormously. (2) "Cowork is easy mode, Claude Code is hard mode" — wrong, they differ by delegation style, not difficulty. (3) "Try all surfaces at once" — wrong, master one first then expand.

Good first tasks by surface:

  • Chat: "Summarize this report in 5 sentences with 3 questions."
  • Projects: "Read this folder and create a one-page background knowledge summary."
  • Cowork: "Read meeting notes and save a weekly brief draft to outputs/."
  • Claude Code: "Read this bug reproduction path, then write a hypothesis and fix plan in plan.md."

Technology products evolve rapidly. To stay grounded, distinguish between volatile details and lasting principles.

📌 Key Concept: The ecosystem is "hot" when four conditions are met simultaneously: (1) user behavior changes — people design workspaces instead of crafting prompts, (2) operational habits change — teams maintain CLAUDE.md as shared contracts, (3) the ecosystem expands — official and community skills, plugins, MCP servers multiply, (4) job roles are redesigned — marketing, tax, PM, design teams use Claude as an automation layer, not just a chatbot.
